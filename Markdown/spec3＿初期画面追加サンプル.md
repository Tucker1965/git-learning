# マスタメンテナンス管理システム 仕様書

## 1. 概要
本システムは、都市マスタ（`scraped_cities`）の管理を行うWebアプリケーションである。未認証ユーザーに対し初期画面（タイトル・概要）を提供し、ログイン認証後にメンテナンス機能へのアクセスを許可する。

## 3. 機能一覧

### 3.1. 初期画面 (Public Landing Page)
* **URL**: `/`
* **仕様**:
    * システムのタイトルと概要説明を表示する。
    * 画面右上に「ログインアイコン」を配置。
    * アイコン押下でログイン画面 (`/login`) へ遷移。

### 3.2. ログイン機能 (Authentication)
* **URL**: `/login`
* **仕様**:
    * `userid`, `password` の入力フォームを提供。
    * 認証成功時：Flaskの `session` を発行し、マスタメンテナンス画面へリダイレクト。
    * 認証失敗時：エラーメッセージを表示し、フォーム画面を維持。

### 3.3. マスタメンテナンス画面 (Maintenance)
* **URL**: `/maintenance`
* **仕様**:
    * **認証制限**: 未認証ユーザーによる直接アクセスを拒否（`@login_required` 等の実装）。
    * **一覧表示**: `scraped_cities` テーブルのデータを表示。
    * **CRUD操作**:
        * **追加**: フォーム入力による新規レコード挿入。
        * **変更**: 既存レコードの編集。
        * **削除**: 削除確認ダイアログ表示後のレコード削除。

## 4. UI/UX 要件

## 5. セキュリティ対策 (OWASP Top 10 準拠)

| 項目 | 対策内容 |
| :--- | :--- |
| **アクセス制御** | 全メンテナンス機能にセッションチェックを実装。 |
| **インジェクション対策** | SQLAlchemyのORM利用によるSQLの自動生成。 |
| **セッション管理** | ログイン成功時のセッション再生成（セッション固定攻撃対策）。 |

## 6. 技術スタック
* **フレームワーク**: Flask
* **ORM**: Flask-SQLAlchemy
* **DB**: MySQL
* **認証管理**: Flask-Login
