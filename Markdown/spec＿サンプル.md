
# マスタメンテナンス機能 実装仕様書

## 1. 概要
本機能は、管理者ユーザーがログイン認証を経て、「都市マスタ（`scraped_cities`）」の一覧表示および追加・更新・削除を行うためのメンテナンス画面へ遷移・操作する仕組みを提供する。

## 2. システム構成
* **言語/フレームワーク**: Python 3.9 / Flask 3.0.0
* **DB**: MySQL 8.0
* **認証**: Flask Session を利用した認証管理

## 4. 機能仕様

### 4.1. ログイン機能 (`/login`)
* **役割**: 管理者認証を行う。
* **仕様**:
    * ユーザー名とパスワードの入力フォームを提供。
    * 認証成功時：セッションを作成し、メインメニュー画面へ遷移。
    * 認証失敗時：エラーメッセージを表示し、ログイン画面に留まる。

### 4.2. メインメニュー (`/menu`)
* **役割**: 各管理機能への入り口。
* **仕様**:
    * 認証済みセッションを確認（未認証時はログイン画面へリダイレクト）。
    * 「都市マスタ管理」へのリンクを配置。
    * 「ログアウト」ボタン（セッション破棄）を配置。

### 4.3. マスタメンテナンス画面 (`/maintenance`)
* **役割**: `scraped_cities` テーブルのCRUD操作。
* **仕様**:
    * **一覧表示**: 全都市情報をテーブル形式で表示。
    * **操作メニュー**:
        * **[追加]**: 新規データ登録フォームへ遷移。
        * **[変更]**: 指定データの編集フォームへ遷移。
        * **[削除]**: JavaScriptの `confirm()` を実行後、該当データをDBより削除。

* **アクセス制御**: 全ての管理機能に `@login_required` デコレータを適用し、直リンクアクセスを遮断する。
* **セキュリティ対策**:
    * SQLインジェクション防止（SQLAlchemyの利用）。
    * セッション固定攻撃対策（ログイン成功時のセッション再生成）。
* **操作性**: 削除時には必ず確認ダイアログを表示し、ユーザーの誤操作を防止する。
